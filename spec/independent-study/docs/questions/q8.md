# Research Question 8

## Q8: What governance latency does schema validation introduce, and at what decision frequency does asynchronous validation become necessary?

<div class="answer-brief">
<p><strong>Answer in brief:</strong> The RGDS reference implementation validates GDR schemas synchronously in CI/CD at human decision speed — phase-gate decisions measured in days and hours. Agentic systems spawn sub-decisions at machine speed. For decisions under 100ms latency requirements, synchronous schema validation introduces unacceptable overhead. The tiered architecture applies: consequential decisions receive full synchronous GDR validation; high-frequency, low-consequence decisions receive lightweight asynchronous log records validated in batch. The governance integrity of consequential decisions is preserved by defining which decision classes require synchronous treatment in the governance covenant. <span class="conf-med">Medium Confidence</span></p>
</div>

---

### The Speed-Governance Tension

GDI's CI/CD integration pattern embeds schema validation in the decision pipeline. Every GDR is validated against the schema before the decision is committed. In the RGDS reference implementation, this adds negligible latency to phase-gate decisions that already take days.[[1]](#cite1)

Agentic AI systems operate at a different timescale. A fraud detection model evaluates millions of transactions per hour. An autonomous network routing agent makes sub-millisecond decisions. A clinical decision support system processes patient queue updates in real time. Synchronous full GDR validation would introduce latency that makes these systems non-functional.

---

### The Tiered Resolution

GDI's confidence threshold model already implements risk-proportional governance: GREEN-zone decisions receive lighter oversight than AMBER or RED-zone decisions. The same proportionality applies to governance latency.

**Synchronous full GDR validation** applies to: decisions above a defined consequence threshold (financial exposure, patient safety impact, regulatory significance); AMBER and RED-zone decisions by definition; any decision in a high-risk AI system under EU AI Act Annex III classification.

**Asynchronous lightweight logging** applies to: high-frequency, low-consequence decisions (individual transaction screening below a dollar threshold; routine content classification; low-stakes recommendations); GREEN-zone decisions in non-regulated verticals; decisions with pre-authorized parameters within a defined governance covenant scope.

**Batch validation** applies to: asynchronous log records, validated against a lightweight schema hourly or daily; anomaly detection on the batch to identify governance drift signals (see Q6).

The critical design principle: the governance covenant defines which decision classes receive which validation tier. An agent operating under a governance covenant that assigns all decisions to the lightweight tier has not been governed — it has been given a governance bypass. The covenant tier assignment must be derived from the decision class risk profile, not from operational convenience.

---

### Quantitative Frame

Financial institutions using policy-as-code have achieved 90%+ reductions in compliance violations from human error.[[2]](#cite2) The latency overhead of policy evaluation at runtime — the closest published analog to GDI schema validation — is measured in milliseconds for simple rule evaluation and tens of milliseconds for complex policy trees. For decisions with latency requirements above 50ms, synchronous full GDR validation is compatible with operational requirements. Below 50ms, asynchronous validation is required.

These are estimates derived from analogous systems. The RGDS reference implementation has not been benchmarked in a high-frequency agentic environment. This is a research gap.

---

### Limitations

The quantitative frame is derived from analogous systems, not from GDI benchmarks. Medium confidence reflects this. The tiered architecture is the right design; the specific latency thresholds require empirical validation in production agentic systems.
