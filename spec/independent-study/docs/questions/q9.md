# Research Question 9

## Q9: For six major frameworks, which specific requirement does GDI satisfy, through which field or mechanism, with what operational evidence?

<div class="answer-brief">
<p><strong>Answer in brief:</strong> GDI satisfies specific, named requirements in five of six frameworks examined, with operational evidence from the RGDS reference implementation. The sixth framework (Oracle OCI AGF) is addressed separately in Q1. The mappings below are specific: each claim names the framework requirement, the GDI field or mechanism that satisfies it, and where operational evidence exists. Theoretical compatibility claims without implementation evidence are flagged as such. <span class="conf-high">High Confidence</span></p>
</div>

### Framework Mappings

#### NIST AI RMF 1.0

| Requirement | GDI Implementation | Evidence |
|-------------|-------------------|----------|
| GOVERN 1.1: Policies, processes, procedures in place | GDR schema defines the governed decision process | RGDS CI/CD validation enforces schema compliance on every commit[[1]](#cite1) |
| GOVERN 2.2: Accountability for AI risk roles and decisions | GDR accountability chain — named human owner, reviewers, approvers | 6 canonical decision examples in RGDS repository demonstrate accountability chain population[[1]](#cite1) |
| MAP 1.1: Context established for AI risk | GDR decision question, options considered, evidence base, and data provenance | Schema-enforced; validated in RGDS test suite (114 tests)[[1]](#cite1) |
| MEASURE 2.5: AI system lifecycle monitoring | GDR's downstream propagation map; CI/CD governance validates on every deployment | GitHub Actions workflow in RGDS repository — 170+ commit history[[1]](#cite1) |

#### ISO/IEC 42001:2023

| Requirement | GDI Implementation | Evidence |
|-------------|-------------------|----------|
| A.6.2.8: Event log requirements for AI systems | GDR is the event log; schema-validated, immutable on commit | RGDS Git history provides tamper-evident audit trail[[1]](#cite1) |
| A.6.1.2: AI risk assessment | GDR risk posture field with explicit residual risk classification | Schema-enforced; residual risk field required[[2]](#cite2) |
| A.6.2.3: Human oversight of AI systems | GDR accountability chain; AI assistance disclosure protocol | Operational in RGDS — `aiassistance` object in every AI-assisted decision log[[1]](#cite1) |

#### EU AI Act 2024/1689

| Requirement | GDI Implementation | Evidence |
|-------------|-------------------|----------|
| Art. 12: Automatic event logging | GDR logged before action executes; CI/CD validation enforces | RGDS GitHub Actions workflow validates before merge[[1]](#cite1) |
| Art. 14: Human oversight capability | Accountability chain requires named human owner; escalation path defined | RGDS canonical examples include all five gate outcomes including escalate[[1]](#cite1) |
| Art. 86: Right to explanation of individual decisions | GDR contains decision question, options evaluated, evidence base, rationale | Theoretical mapping — not yet tested in EU enforcement context[[2]](#cite2) |

#### ARAF v3.0

| Requirement | GDI Implementation | Evidence |

| Reconstructability principle | GDR captures contemporaneous governance record; Git-versioned | RGDS demonstrates 2-minute retrieval of complete decision record vs. 2-3 week reconstruction[[1]](#cite1) |
| Chain-complete evidence across Decision Supply Chain | GDR data provenance fields span structured and unstructured sources | Schema-enforced; completeness classification per evidence item[[2]](#cite2) |

#### AIGN OS

| Requirement | GDI Implementation | Evidence |
|-------------|-------------------|----------|
| Layer 4: Governance implementation | GDR is the governed artifact Layer 4 requires | Theoretical mapping; AIGN OS does not specify artifact schema[[3]](#cite3) |
| Layer 1: Trust infrastructure audit evidence | GDRs in Git provide audit-ready evidence artifacts | RGDS repository provides operational demonstration[[1]](#cite1) |

---

### Limitations

The EU AI Act Article 86 mapping is theoretical. The Article requires explanation of individual decisions to affected persons. GDI's GDR contains the information that explanation requires. Whether a GDR as currently specified satisfies the Article in enforcement context depends on regulatory interpretation not yet available (enforcement begins August 2026).

The AIGN OS Layer 4 mapping is theoretical. AIGN OS defines the requirement; it does not specify the artifact. GDI provides the artifact. The mapping is valid as a compatibility claim, not as a verified implementation.
