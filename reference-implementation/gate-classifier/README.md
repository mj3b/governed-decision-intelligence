# Gate Classifier — GDI Reference Implementation

Decision-layer gate classification extending Microsoft's [Agent Governance Toolkit (AGT)](https://github.com/microsoft/agent-governance-toolkit).

---

## What This Is

AGT governs what an agent **does** — policy enforcement at the action layer (allow/deny before execution).

The gate classifier governs what **category of decision** the agent was making — what level of human deliberation does this decision require?

These are adjacent questions, not identical ones. Both are necessary for institutional accountability. A policy engine that allows an action does not determine whether that action required human review before proceeding. A gate classifier that flags elevated review does not determine whether the action was permitted. Together they address the two questions that matter under audit: was this action authorized, and was the decision to take it appropriately governed?

```
Agent → tool call intent
  → PolicyEngine.evaluate()        [AGT: allow/deny]
  → GateClassifier.classify()      [GDI: Gate 1–4 classification]
  → GateRecord.write()             [Immutable artifact before execution]
  → Action executes (or escalates)
```

The gate record is written before execution. An artifact written after the fact is a narrative. An artifact written before execution is evidence.

---

## The Four Gates

| Gate | Name | Human Deliberation |
|------|------|--------------------|
| Gate 1 | Routine Execution | None at decision time. Automated documentation. |
| Gate 2 | Documented Delegation | None at decision time. Upstream delegation invoked and recorded. |
| Gate 3 | Elevated Review | Mandatory. A designated reviewer must acknowledge the artifact. |
| Gate 4 | Hard Escalation | Execution halts. Human authority required before proceeding. |

Gate classification is a normative act, not an observational one. It must be defined by the institution and applied by the governance layer — not generated by the model being governed. Self-certification is not governance.

## Usage

```python
from gate_classifier import GateClassifier, Gate

classifier = GateClassifier()
record = classifier.evaluate(
    policy_allowed=True,
    policy_confidence_threshold=0.8,
    agent_id="claims-agent-001",
    tool_name="database_write",
    tool_args={"table": "claims_decisions"},
    confidence_score=0.85,
)

# record.gate == Gate.ELEVATED_REVIEW
# record.record_hash  — SHA-256 sealed before execution
# record.gate_rationale  — institutional explanation
# record.reasoning_reconstruction  — plain-language derivation
```

## Integration with AGT

The `GateClassificationMixin` integrates with any `BaseIntegration` subclass from AGT:

```python
from agent_os.integrations.base import BaseIntegration, GovernancePolicy
from gate_classifier import GateClassificationMixin, GateClassifier

class GovernedIntegration(GateClassificationMixin, BaseIntegration):
    def __init__(self, policy, classifier=None):
        super().__init__(policy=policy)
        self.setup_gate_classifier(classifier)

    def pre_execute(self, ctx, input_data):
        allowed, reason = super().pre_execute(ctx, input_data)
        self.classify_after_policy_check(
            ctx=ctx,
            policy_allowed=allowed,
            tool_name=getattr(input_data, 'tool_name', 'unknown'),
            tool_args=getattr(input_data, 'arguments', {}),
        )
        return allowed, reason
```

## What the Gate Record Contains

Each gate record is written before execution and contains:

- `gate` — the classification outcome (Gate 1 through Gate 4)
- `gate_rationale` — institutional explanation of why this gate applies
- `confidence_score` — against the policy threshold
- `reasoning_reconstruction` — plain-language derivation from structured fields only (no model-generated prose)
- `record_hash` — SHA-256 of immutable fields at classification time
- `escalation_required` / `escalation_trigger` — populated when Gate 4 fires

The hash enables tamper detection. Any post-hoc modification of the record produces a hash mismatch. The record is evidence of what the governance layer knew and decided before execution.
