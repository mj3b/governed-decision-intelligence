# Gate Taxonomy: Decision-Level Accountability for Agent Governance

**Author:** Mark Julius (`mj3b`)  
**Status:** Draft v0.1  
**Repo:** `mj3b/governed-decision-intelligence`  
**License:** Apache 2.0

---

## What This Is

This specification defines a gate taxonomy for classifying agent decisions at the moment of policy evaluation.

It extends Microsoft's Agent Governance Toolkit (AGT) without modifying it. AGT governs what an agent *does*. This extension classifies what kind of *decision* the agent was making when it attempted that action.

The distinction matters for accountability. A policy engine answers: *is this action permitted?* A gate classifier answers: *what level of human deliberation does this decision require?*

Those are adjacent but not identical questions. Both are necessary for institutional accountability.

---

## The Problem This Solves

AGT's `PolicyEngine.evaluate()` returns an allow or deny result with a reason code. That output is sufficient for security enforcement. It is insufficient for accountability because it treats all allowed decisions as equivalent.

A decision made with high confidence against one clear alternative is not equivalent to a decision made with marginal confidence against five competing alternatives. Both may clear the policy engine. Only a gate classifier reveals the governance weight each decision carries.

Without gate classification, audit trails record *what happened* but not *what category of decision* produced it. That gap is where institutional accountability collapses into technical compliance.

---

## Definition: What a Gate Is

A gate is not a threshold. A threshold is a number. A gate is a governance classification that carries institutional weight.

A gate answers one question: **what level of human deliberation does this decision require before or after execution?**

Gate classification is a normative act, not an observational one. It evaluates the decision against a governance standard. That is why it must be defined by the institution, not generated by the model being governed. Self-certification is not governance.

---

## The Four Gates

### Gate 1: Routine Execution

**Definition:** The decision falls within established parameters, has clear precedent, and carries low consequence if wrong.

**Characteristics:**
- Action type is explicitly pre-authorized in policy
- Agent confidence is high and unambiguous
- No novel conditions present
- Consequence of error is recoverable

**Human deliberation required:** None at decision time. Automated documentation only.

**Audit artifact:** Timestamped log entry recording gate classification, action type, agent identity, and policy rule matched.

---

### Gate 2: Documented Delegation

**Definition:** The decision exceeds routine parameters or carries meaningful consequence, but falls within pre-authorized scope defined upstream by a human authority.

**Characteristics:**
- Action type is authorized but carries elevated consequence
- Agent confidence is high but the action class is non-trivial
- Human deliberation occurred upstream when scope was defined
- The delegation chain is traceable

**Human deliberation required:** Not at decision time. The artifact records that a specific upstream delegation was invoked, by whom, and under what authority.

**Audit artifact:** Gate classification plus delegation reference. The record identifies the human authority whose prior deliberation covers this decision.

---

### Gate 3: Elevated Review

**Definition:** The decision is novel, consequential, or sits at the boundary of authorized scope. A human must review the artifact before or immediately after execution.

**Characteristics:**
- Action type is permitted but the specific conditions are not clearly precedented
- Agent confidence is present but the decision boundary is close
- Consequence of error may not be fully recoverable
- A human reviewer can materially improve the outcome

**Human deliberation required:** Mandatory. A designated reviewer must acknowledge the artifact. This is not a veto by default, but it is a required deliberative moment.

**Audit artifact:** Gate classification, reasoning state at decision time, confidence assessment, conditions that triggered elevated review, and reviewer acknowledgment with timestamp.

---

### Gate 4: Hard Escalation

**Definition:** The decision exceeds authorized scope, involves irreversible action, or presents conditions the system cannot classify with sufficient confidence.

**Characteristics:**
- Action type approaches or exceeds defined authorization boundaries
- Consequence of error is irreversible or materially significant
- Confidence is insufficient for autonomous execution
- No precedent covers the specific conditions

**Human deliberation required:** Execution stops. Human authority must explicitly authorize before the action proceeds. This is not a notification. It is a hard stop.

**Audit artifact:** Gate classification, full reasoning state, explicit escalation trigger, human authorization record with identity and timestamp before execution resumes.

## Insertion Point in AGT Architecture

The gate classifier wraps AGT's `PolicyEngine.evaluate()` call. It does not modify the policy engine's allow/deny logic. It adds classification after the policy result is returned, before the calling agent acts on it.

```
Agent → tool call intent
  → PolicyEngine.evaluate()     [AGT: allow/deny]
  → GateClassifier.classify()   [This extension: Gate 1–4]
  → GateRecord.write()          [Immutable artifact before execution]
  → Action executes (or escalates)
```

The gate record is written *before* execution. An artifact written after the fact is a narrative. An artifact written before execution is evidence. That distinction is the entire accountability claim.

---

## Why Classification Must Be External to the Model

The gate taxonomy must be defined by the institution and applied by the governance layer, not generated by the model being governed.

A model that classifies its own decisions is producing self-assessment, not governance. Self-assessment is useful input. It is not an accountability artifact.

The gate classifier in this extension applies institutional rules to the decision context. The model's confidence scores and reasoning state are inputs to that classification. They are not the classification itself.

---

## Relationship to Microsoft's AGT

This extension is complementary to AGT, not competitive with it.

AGT governs the action layer: what an agent is permitted to do.  
This extension governs the decision layer: what category of decision the agent was making.

Both layers are necessary. A governed action without a classified decision produces a compliant audit trail. A classified decision without governed action produces accountability without enforcement. The full governance stack requires both.
